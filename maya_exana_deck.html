The Seed Is the Exam — Unique, Yet Reproducible

A student ID is hashed (xmur3) into a 128-bit state that seeds an sfc32 PRNG. Same ID always rebuilds the same paper (auditable); different IDs get different papers (uncheatable).
STUDENT A seed: NEET-100482

key: B
Q2. CI on 7000 @ 12%, 3 yr
key: D
Q3. v = u + at (6, 4, 5)
key: A
VS
STUDENT B seed: NEET-100483
Q1. 251 km in 3 h → speed?
key: C
Q2. CI on 17000 @ 12%, 3 yr
key: B
Q3. v = u + at (3, 6, 6)
key: B
A's leaked answer key (B, D, A ...) scores ~random on B's paper. Sharing is pointless.
Why this matters
  • 128-bit seed space → no practical collisions even at national scale.
  • No question text stored on a server → nothing to leak.
  • Re-buildable → Any paper is deterministically re-buildable from its ID for audit, appeal, or re-grade.
